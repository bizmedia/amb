# ADR-008: PostgreSQL Row-Level Security (RLS) для изоляции по тенанту/проекту

Статус: Принято  
Дата: 2026-01-28

## Контекст

Фильтрация на уровне приложения (например, `WHERE project_id = ...`) необходима, но в мультитенантной системе недостаточна. Нужна защита в глубину, чтобы:

- Пропущенный фильтр не приводил к утечке данных между тенантами.
- Будущие изменения по умолчанию оставались безопасными.

В репозитории используется Prisma с пулом соединений; контекст на уровне сессии нужно задавать аккуратно.

## Решение

Включить в PostgreSQL Row-Level Security (RLS) на всех таблицах, привязанных к проекту.

Подход к политикам:

- Для каждого запроса задаётся контекст сессии БД:
  - `app.tenant_id`
  - `app.project_id`
- Политики RLS для `SELECT/UPDATE/DELETE` используют `current_setting('app.tenant_id', true)` и `current_setting('app.project_id', true)` для ограничения видимости строк.
- Для `INSERT/UPDATE` используется `WITH CHECK`, чтобы запретить запись строк вне контекста запроса.

Интеграция с Prisma:

- Весь доступ к БД в `apps/api` выполняется внутри транзакции, в которой вызываются `SET LOCAL app.tenant_id = ...; SET LOCAL app.project_id = ...;`.
- Доменные сервисы получают Prisma-клиент в контексте этой транзакции, чтобы все запросы использовали один и тот же контекст сессии.

Административные задачи (миграции, глобальное обслуживание):

- Выполняются под отдельной ролью БД с `BYPASSRLS` (или изолированным соединением), чтобы не усложнять обход политик в коде приложения.

## Рассмотренные альтернативы

1) Только scoping на уровне приложения (без RLS)

- Плюсы: проще, меньше зависимости от Postgres.
- Минусы: выше риск утечки данных; слабая защита в глубину.

2) Отдельная БД на тенанта

- Плюсы: максимальная изоляция.
- Минусы: операционные затраты; усложняет онбординг и DX; не требуется на начальном масштабе.

## Последствия

- Нужны единообразные паттерны доступа к БД в рамках запроса (обёртка транзакции).
- Дополнительная работа по миграциям (включение RLS и политик).
- Повышается безопасность и снижается вероятность инцидентов между тенантами.
