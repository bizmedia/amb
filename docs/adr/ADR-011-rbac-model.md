# ADR-011: Модель RBAC (tenant-admin, project-admin, reader)

Статус: Принято  
Дата: 2026-01-28

## Контекст

Мультитенантная система требует ролевого управления доступом (RBAC) для прав на уровне тенантов и проектов.

## Решение

Ввести **модель RBAC с тремя ролями**:

1. **tenant-admin**
   - Полный доступ к тенанту и всем проектам
   - Может создавать и управлять проектами
   - Может выдавать project-токены
   - Может управлять настройками тенанта
   - Может назначать роли пользователям

2. **project-admin**
   - Полный доступ к конкретному проекту
   - Может управлять настройками проекта
   - Может выдавать project-токены для своего проекта
   - Может просматривать и управлять агентами, тредами, сообщениями в проекте
   - Не имеет доступа к другим проектам и настройкам тенанта

3. **reader**
   - Только чтение в рамках проекта
   - Может просматривать агентов, треды, сообщения
   - Не может изменять данные
   - Не может выдавать токены
   - Не имеет доступа к настройкам тенанта

### Детализация прав
- **MVP:** на уровне ролей (грубая гранулярность)
- **Будущее:** детальные права по ресурсу/действию (при необходимости)

## Рассмотренные альтернативы

1) Детальные права (permission-based)
- Плюсы: максимальная гибкость
- Минусы: сложно реализовать и поддерживать, избыточно для MVP

2) Только две роли (admin, reader)
- Плюсы: проще
- Минусы: нельзя разделить админа тенанта и админа проекта

3) Более дробные роли (tenant-reader, project-reader и т.д.)
- Плюсы: больше гибкости
- Минусы: избыточная сложность для MVP

## Последствия

- В JWT должен быть claim `roles` (массив)
- API endpoints проверяют роли:
  - Операции на уровне тенанта требуют `tenant-admin`
  - Операции на уровне проекта — `project-admin` или `reader` (в зависимости от действия)
- UI Dashboard должен учитывать роли
- Project-токены наследуют контекст проекта (без ролей пользователя)

## Реализация

### Структура claims в JWT
```typescript
{
  sub: "user",
  tenantId: string,
  roles: ["tenant-admin"] | ["project-admin"] | ["reader"],
  projectId?: string, // для токенов в контексте проекта
  // ...
}
```

### Матрица прав

| Действие | tenant-admin | project-admin | reader |
|----------|--------------|---------------|--------|
| Создать проект | ✅ | ❌ | ❌ |
| Управлять проектом | ✅ | ✅ (свой) | ❌ |
| Выдать project-токен | ✅ | ✅ (свой) | ❌ |
| Просмотр данных проекта | ✅ | ✅ (свой) | ✅ (свой) |
| Изменение данных проекта | ✅ | ✅ (свой) | ❌ |
| Управление тенантом | ✅ | ❌ | ❌ |

## Возможные улучшения

- Пользовательские роли с детальными правами
- Наследование прав
- Временный доступ (роли с ограничением по времени)
- Аудит изменений прав
