# UX Design System: Agent Message Bus Dashboard

**Версия:** 1.0  
**Дата:** 2026-03-17  
**Роль:** UX Agent  
**Статус:** Рекомендации для реализации

**Визуальный референс:** стиль `developer editorial`: нейтральная oklch-палитра с акцентами, радиус 0.75rem+, тени elevation, контрастная типографическая пара (Unbounded для заголовков, IBM Plex Sans/Mono для контента и кода), карточные панели и шапка с фиксированной высотой.

---

## 1. Цели интерфейса и задачи посетителя

| Задача посетителя | Решение в UI |
|-------------------|--------------|
| Управлять агентами и тредами в одном месте | Трёхколоночный layout: агенты → треды → контент (messages/inbox/dlq) |
| Быстро переключаться между контекстами | Выбор агента/треда + вкладки; Command Palette (⌘K) |
| Видеть состояние доставки и ошибок | Badges (inbox/DLQ), статусы сообщений, индикатор SSE |
| Создавать треды и отправлять сообщения | Кнопка «Новый тред», форма в Thread view |
| Восстанавливать failed-сообщения | Вкладка DLQ с Retry one / Retry all |
| Архивировать/закрывать и удалять сущности | Единый паттерн: dropdown по строке → подтверждение для удаления |

Учтено расширение: **удаление тредов, агентов и т.д.** — все такие действия следуют одному паттерну (см. раздел 5).

---

## 2. Визуальная и компонентная консистентность

### 2.1 Стек и ограничения

- **Компоненты:** shadcn/ui (Button, Badge, Input, Tabs, Dialog, DropdownMenu, ScrollArea, Tooltip).
- **Стиль:** локальный dev-инструмент, без брендинга; нейтральная палитра (primary/muted/destructive).
- **Обновления:** polling (и при наличии — SSE); без WebSocket в текущем scope.

### 2.2 Единые отступы и размеры

| Элемент | Значение | Использование |
|---------|----------|----------------|
| Паддинг панели заголовка | `p-4` | Заголовки списков (Agents, Threads), секции |
| Паддинг контента списка | `p-2` | Внутренний отступ ScrollArea |
| Высота строки списка | `py-2.5 px-3` | Карточка агента/треда |
| Высота поиска | `h-8` | Input в сайдбарах |
| Иконки в кнопках | `size-3.5`–`size-4` | Единый размер в списках и хедере |
| Радиус карточек | `rounded-lg` | Строки списков, карточки контента |

### 2.3 Типографика

- **Заголовок приложения:** `text-lg font-semibold tracking-tight`.
- **Заголовок панели (Agents/Threads):** `font-semibold text-sm`.
- **Заголовок строки (имя агента, название треда):** `font-medium text-sm`.
- **Вторичный текст (роль, дата):** `text-xs` или `text-[11px]`, `text-muted-foreground`.
- **Badge счётчиков:** `text-[10px]` или `text-xs`.
- **Текст сообщений в Thread view:** `IBM Plex Sans`, `text-[0.95rem]`, `leading-7` для комфортного чтения длинных технических сообщений.
- **Код/JSON в сообщениях:** `IBM Plex Mono`; переносы обязательны, чтобы длинные строки не ломали layout.

### 2.4 Состояния строк списка

- **Default:** фон не выделен.
- **Hover:** `hover:bg-muted/50`.
- **Selected:** `bg-primary/10 ring-1 ring-primary/20`.
- **Disabled / temp:** `opacity-60`, `cursor-not-allowed` при необходимости.

### 2.5 Кнопки в панелях

- **Refresh:** `variant="ghost"`, `size="icon"`, класс `size-7` — только иконка.
- **Primary action (например, «Новый тред»):** иконка + текст в DialogTrigger или отдельная кнопка `size="icon"` в заголовке.
- **Действия по строке:** показывать по hover через `DropdownMenu`, триггер `size-6` или `size-5`, `opacity-0 group-hover:opacity-100`.

### 2.6 Иерархия шапки (Dashboard header)

Шапка не должна быть перегружена: действия делятся на **первостепенные** (в баре) и **второстепенные** (в выпадающем меню «Ещё»).

| Приоритет | Действия | Где показывать | Обоснование |
|-----------|----------|----------------|-------------|
| **Первостепенные** | Выбор проекта (ProjectSwitcher) | В шапке | Контекст работы, частая смена. |
| | Индикатор SSE (connected/reconnecting) | В шапке, компактно | Важно для понимания «живости» данных. |
| | Command Palette (⌘K) | В шапке, одна кнопка | Главная точка входа для навигации и действий. |
| | Переключатель темы (светлая/тёмная) | В шапке, иконка | Часто переключают; оставлен в панели. |
| **Второстепенные** | API Docs, Help | В меню «Ещё» | Справка, не каждый день. |
| | Язык (Locale) | В меню «Ещё» | Настраивается редко. |
| | Выход (Log out) | В меню «Ещё», с разделителем | Редкое действие; визуально отделить от остального. |

**Визуальные правила шапки:**

- **Левый блок:** логотип + название приложения (и опционально subtitle). Без кнопок.
- **Центр/право:** только первостепенные элементы в одну линию: `[ProjectSwitcher] [SSE] [Commands ⌘K]`.
- **Индикатор SSE:** только точка (зелёная/жёлтая) + tooltip с текстом «SSE connected» / «Reconnecting…»; подпись «Real-time» в бар не выносить или показывать только в tooltip.
- **Переключатель темы:** в шапке, одна кнопка `variant="ghost"` `size="icon"` (Sun/Moon), tooltip с подписью темы.
- **Меню «Ещё» (More):** одна кнопка `variant="ghost"` `size="icon"` с иконкой `MoreVertical`, при клике — DropdownMenu: API Docs, Help, разделитель, Language (submenu), разделитель, Log out.

Итоговая схема шапки:

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ [Logo] Title (subtitle)     [Project ▼]  (•)  [⌘ K]  [☀/☽]       [⋮ More]   │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↑              ↑      ↑              ↑
                               SSE tooltip   Commands  Theme      Dropdown:
                                                                 API Docs, Help
                                                                 ————————
                                                                 Language
                                                                 ————————
                                                                 Log out
```

При добавлении новых глобальных действий: если действие не ежедневное — помещать в «Ещё», а не в бар.

## 3. Паттерны экранов

### 3.1 Список (Agents, Threads)

```
┌─────────────────────────────────────┐
│ [Icon] Title        [Badge] [⟳][+] │  ← заголовок + счётчик + refresh + add (если есть)
├─────────────────────────────────────┤
│ [🔍] Search...                      │
│ [All] [Open] [Closed] [Archived]    │  ← только для Threads (фильтр)
├─────────────────────────────────────┤
│ ┌─────────────────────────────────┐ │
│ │ [•] Item name          [Badge] ⋮ │ │  ← строка: иконка статуса, название, badge, menu
│ │     secondary text               │ │
│ └─────────────────────────────────┘ │
│ ┌─────────────────────────────────┐ │
│ │ ...                             │ │
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

- **Empty:** центрированный блок с иконкой (`opacity-20`), заголовок, подпись, при необходимости CTA (например «Создать тред» / «Очистить поиск»).
- **Loading:** центрированный `Loader2Icon` + текст «Загрузка…».
- **Error:** иконка ошибки, текст ошибки, кнопка «Повторить».

### 3.2 Главная область (Messages / Inbox / DLQ)

- **Messages:** заголовок треда (или placeholder), группировка по дате, карточки сообщений, форма отправки (если выбран тред и агент).
- **Inbox:** список сообщений в inbox выбранного агента с действиями ACK и т.п.
- **DLQ:** заголовок с «Retry all», список сообщений с раскрытием payload и «Retry» по одному.

Единое правило: **все списки сущностей** (агенты, треды, сообщения, DLQ) используют один визуальный язык: карточка строки, вторичный текст, при необходимости dropdown действий.

### 3.3 Модальные окна

- **Создание (например, тред):** Dialog с заголовком, описанием, полем ввода, Cancel + Submit. Submit disabled, пока нет валидного ввода.
- **Подтверждение деструктивного действия:** AlertDialog или Dialog с явным заголовком («Удалить тред?»), кратким описанием последствий, Cancel + Destructive кнопка («Удалить»).

---

## 4. Состояния (empty / loading / error)

| Состояние | Визуал | Поведение |
|-----------|--------|-----------|
| **Empty (нет данных)** | Иконка приглушённая, заголовок, подпись, опционально CTA | Кнопка ведёт к созданию или сбросу фильтра |
| **Loading** | `Loader2Icon` + текст «Загрузка…» | Без кликабельных действий |
| **Error** | Иконка ошибки (AlertCircle), текст ошибки | Кнопка «Повторить» / «Try again» вызывает refetch |
| **Empty после фильтра** | Как empty, но подпись «Ничего не найдено» + кнопка «Сбросить фильтры» | Сброс поиска и фильтра |

Во всех списках (Agents, Threads, Inbox, DLQ) использовать эти три состояния консистентно.

---

## 5. Деструктивные и управляющие действия (удаление и расширения)

Цель: один предсказуемый паттерн для **удаления тредов**, **удаления агентов** и будущих сущностей (например, bulk delete).

### 5.1 Где показывать действия

- **По одной сущности:** в строке списка, через **DropdownMenu** (триггер — иконка «три точки» или «шестерёнка»), видим по hover строки.
- **Bulk (будущее):** при выборе нескольких строк — панель над списком «Выбрано N» с кнопкой «Удалить выбранные» и т.п.

### 5.2 Состав пунктов меню

- **Не деструктивные:** Open / Close / Archive и т.д. — без разделителя.
- **Разделитель** (`DropdownMenuSeparator`) перед деструктивным пунктом.
- **Деструктивный пункт:** «Удалить» с иконкой Trash, классы `text-destructive focus:text-destructive`.

Пример для треда (уже так в коде):

- Open / Close / Archive (по контексту)
- ---
- Удалить

Пример для агента (при добавлении API):

- Перейти в треды агента (опционально)
- ---
- Удалить агента

### 5.3 Подтверждение удаления

- Всегда **отдельный Dialog** с:
  - заголовком: «Удалить [сущность]?» (например, «Удалить тред?»);
  - кратким описанием последствий (например, «Сообщения треда будут удалены без возможности восстановления.»);
  - кнопками: «Отмена» (outline), «Удалить» (variant destructive).
- Не выполнять удаление без подтверждения (никакого «один клик удалил»).

### 5.4 После удаления

- Закрыть диалог, убрать сущность из списка (оптимистично или после успешного ответа API).
- Если удалена выбранная сущность (например, текущий тред): снять выбор и выбрать другую подходящую (например, следующий тред в списке) или оставить пустое состояние в правой панели.

## 6. Горячие клавиши

Сохранять и расширять единый слой:

| Shortcut | Действие |
|----------|----------|
| ⌘K | Command Palette |
| ⌘N или заданный | Новый тред |
| ⌘R или F5 | Обновить данные |
| ⌘1 / ⌘2 / ⌘3 | Переход на вкладки Messages / Inbox / DLQ (если реализовано) |
| / или ⌘F | Фокус в поиск (уже через Command Palette) |

Все shortcuts должны быть указаны в Command Palette и при необходимости в подсказках (tooltip).

---

## 7. Оптимистичный UI

- **Создание треда:** можно показать временный тред в списке (например, `id: temp-…`) с пометкой загрузки и после ответа API заменить на реальный.
- **Смена статуса (open/closed/archived):** обновить UI сразу; при ошибке — откатить и показать ошибку (toast или inline).
- **Удаление:** убрать строку из списка после подтверждения и успешного запроса; при ошибке — оставить в списке и показать ошибку.
- **Retry (DLQ):** кнопка «Retry» переходит в loading (spinner), по успеху сообщение исчезает из списка.

Не делать оптимистичное удаление без подтверждения: сначала диалог, потом запрос, потом обновление списка.

---

## 8. Группировка сообщений

В Thread view сохранять группировку по дате (уже есть):

- Заголовок группы: дата в формате «weekday, day month year».
- В группе — карточки сообщений в хронологическом порядке.

Единый вид карточки сообщения: отправитель, время, статус (pending/delivered/ack/dlq), тело (текст/JSON). При расширении UI сообщений не менять общую структуру групп.

---

## 9. Чеклист консистентности для новых фич

При добавлении **удаления агентов**, **bulk delete** или других сущностей:

- [ ] Действие «Удалить» в DropdownMenu по строке, с разделителем и `text-destructive`.
- [ ] Диалог подтверждения с заголовком, описанием и кнопками Отмена / Удалить.
- [ ] Одинаковые отступы и размеры с существующими списками (см. п. 2.2).
- [ ] Empty/Loading/Error по одному шаблону (п. 4).
- [ ] Обновление выбора: если удалён выбранный элемент — снять выбор или выбрать другой.
- [ ] При bulk: панель «Выбрано N» и одна кнопка удаления с одним общим диалогом подтверждения.

## 10. Краткая схема экранов

```
┌──────────────────────────────────────────────────────────────────────────┐
│ [Logo] Dashboard         [Project ▼]  (•)  [⌘ K]              [⋮ More]        │
├────────────┬────────────┬───────────────────────────────────────────────┤
│  AGENTS    │  THREADS   │  Messages | Inbox | DLQ                        │
│  [🔍]      │  [🔍]      ├───────────────────────────────────────────────┤
│  [All|…]   │  [Filters] │                                               │
│  ───────── │  ───────── │   Thread view / Inbox list / DLQ list          │
│  • Agent 1 │  • Thread A│   + message composer (Messages)              │
│  • Agent 2 │  • Thread B│   + Retry actions (DLQ)                       │
│  …         │  …         │                                               │
│            │  [+ New]   │                                               │
└────────────┴────────────┴───────────────────────────────────────────────┘
```

Все списки (Agents, Threads, сообщения, DLQ) следуют одному паттерну строки и состояний; удаление и прочие управляющие действия — через dropdown и подтверждение по п. 5.

---

**Итог:** профессиональный и консистентный интерфейс достигается за счёт единых отступов, типографики, состояний списков и единого паттерна деструктивных действий (меню по строке + диалог подтверждения). Расширение на удаление агентов и другие сущности делается без смены визуального языка — только добавлением пунктов в меню и новых диалогов по тому же шаблону.
