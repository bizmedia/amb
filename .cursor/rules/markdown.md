# Правило: оформление Markdown

При создании и правке любых файлов **.md** в проекте используйте продвинутые возможности Markdown и уместные Emoji, чтобы текст был **максимально читабельным**.

## Продвинутые возможности Markdown

- **Структура:** иерархия заголовков (H1–H6), маркированные и нумерованные списки, таблицы, цитаты (`>`), разделители (`---`).
- **Выделение:** **жирный**, *курсив*, `инлайн-код`, [ссылки с понятным текстом](url).
- **Таблицы** — для сравнений, справочников параметров, чек-листов.
- **Блоки кода** — с указанием языка (` ```bash `, ` ```json `, ` ```typescript ` и т.д.) для подсветки.
- **Сворачиваемые блоки** — `<details>` / `<summary>` для длинных пояснений, логов, опциональных шагов.
- **Чеклисты** — `- [ ]` и `- [x]` для шагов, требований, критериев приёмки.

## Emoji

Использовать **умеренно**, чтобы улучшить навигацию и восприятие, не перегружая текст. Примеры:

- 📋 списки, чек-листы, содержание
- ✅ готово, выполнено, принято
- ⚠️ предупреждение, важно
- 🔧 настройка, конфигурация
- 📁 структура, каталоги
- 🚀 быстрый старт, запуск
- 📖 документация, справка

Не злоупотреблять: один эмодзи на заголовок или пункт списка обычно достаточно.

## Цель

Текст в .md должен быть легко сканировать глазами, быстро находить нужный раздел и понимать структуру документа.
